# Glossary

- **Instance / gateway:** An OpenClaw process with its own data volume such as `~/.openclaw` inside the container. Each user, role, or assistant persona can have a separate instance.
- **Workspace:** The directory inside the volume that stores `AGENTS.md`, `SOUL.md`, `USER.md`, `MEMORY.md`, `memory/*.md`, and related files.
- **Gateway sandbox:** The inside of the OpenClaw container. It is not the host server. Host paths such as `/srv/...` do not exist there unless they are explicitly mounted.
- **Agent Executor (pattern):** An HTTP service on the same Docker network as OpenClaw that executes commands on the host, or in other containers through the Docker socket, under a strict allowlist. OpenClaw calls it as a tool or skill.
- **Daily memory:** Files under `workspace/memory/YYYY-MM-DD*.md`, written by a human operator, the assistant, or an automation during the day.
- **Nightly memory:** A file such as `YYYY-MM-DD-nightly.md` generated by an external job, for example n8n, from logs and structured context.
- **`latest.md`:** A pointer, implemented as a symlink or copy, to the most useful recent memory file so the assistant can read recent context without guessing file names.
- **Long-term memory (`MEMORY.md`):** Stable facts curated manually or promoted by a conservative automation step.
- **Native semantic memory:** A local index, for example SQLite plus embeddings, that OpenClaw uses for retrieval and recall inside a single instance.
- **Shared sync:** Copying only the files that should remain identical across instances, such as a shared capability appendix, without touching identity or per-instance memory.
